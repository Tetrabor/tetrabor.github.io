What Is Global Variables Administration?

Global Variables Administration provides a uniform way to create and reference global variables across all applications at your environment levels. A global variable is an identifier (or key) paired to a stored value, or a key/value pair.

An example is when using envUrl as the key to reference your environment's URL. Your environment URL then acts as the value. Once paired, you can reference the key anywhere in your application. This separation of the key from its value lets you use your key independent of the data it represents. For example, your key can represent one stored value in your UAT environment level and another at your Production level. That way, your values resolve dynamically based on your application's current environment level, and automatically populate in your current environment level.
